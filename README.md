# CyberMem

AI-powered personal memory system based on [OpenMemory](https://github.com/CaviraOSS/OpenMemory) with production-grade DevOps infrastructure and per-client monitoring.

## Features

- 🧠 **OpenMemory** - Multi-sector cognitive memory engine
- 📊 **Per-Client Metrics** - Track writes/reads/storage by client_id
- 🚀 **Multi-Platform** - Deploy to local/Raspberry Pi/VPS with single command
- 📈 **Observability** - Prometheus + Grafana + Alertmanager
- 🔒 **Security** - Encrypted data at rest/transit
- ☸️ **Production-Ready** - Helm charts, resource limits, HA support

## Quick Start

### Prerequisites

- **Local:** Docker, Docker Compose
- **RPi:** Ansible
- **VPS:** Kubernetes (k3s/k3d), Helm, kubectl

### 1. Clone Repository

```bash
git clone --recursive https://github.com/mikhailkogan/cybermem.git
cd cybermem
```

### 2. Deploy

#### Local Development

```bash
# Copy environment file
cp .env.example .env.local

# Deploy
./deploy.sh --target local

# Access
# - OpenMemory:  http://localhost:8080
# - Grafana:     http://localhost:3000 (admin/admin)
# - Prometheus:  http://localhost:9090
```

#### Raspberry Pi

```bash
# Configure Ansible inventory
cat > ansible/inventory/hosts.ini <<EOF
[rpi]
192.168.1.100 ansible_user=pi
EOF

# Deploy
./deploy.sh --target rpi
```

#### VPS (Kubernetes)

```bash
# Create k3s cluster (or use existing)
k3d cluster create cybermem --port 8080:80@loadbalancer

# Edit values
nano charts/cybermem/values-vps.yaml
# Update: openaiApiKey, passwords, domain

# Deploy
./deploy.sh --target vps
```

## Architecture

```
┌─────────────────────────────────────────────────────────┐
│  Client Requests (Authorization: Bearer <client_id>)    │
└──────────────────┬──────────────────────────────────────┘
                   ↓
┌──────────────────────────────────────────────────────────┐
│  Traefik (Reverse Proxy + Access Logs)                  │
│  - Logs client_id from Authorization header             │
│  - JSON access logs → /var/log/traefik/access.log       │
└──────────────┬───────────────────────────┬───────────────┘
               ↓                           ↓
┌──────────────────────────┐   ┌──────────────────────────┐
│  Vector                  │   │  OpenMemory              │
│  - Parses Traefik logs   │   │  - Stores memories       │
│  - Extracts client_id    │   │  - Writes user_id to DB  │
│  - Generates metrics:    │   └──────────┬───────────────┘
│    • requests_total      │              ↓
│    • duration_seconds    │   ┌──────────────────────────┐
│  - Exports :9091/metrics │   │  PostgreSQL / SQLite     │
└──────────────┬───────────┘   └──────────┬───────────────┘
               ↓                           ↓
┌──────────────────────────┐   ┌──────────────────────────┐
│  Prometheus              │   │  postgres_exporter       │
│  - Scrapes Vector        │←──│  - Per-client counts     │
│  - Scrapes PG exporter   │   │  - Sector breakdowns     │
└──────────────┬───────────┘   └──────────────────────────┘
               ↓
┌──────────────────────────────────────────────────────────┐
│  Grafana Dashboards                                      │
│  - Writes/reads per client                               │
│  - Total memories stored                                 │
│  - Request latency p95                                   │
│  - Embeddings provider latency                           │
└──────────────────────────────────────────────────────────┘
```

## Metrics

### Application Metrics (Vector)

```promql
# Total requests by client
openmemory_requests_total{client="client_123", method="POST", endpoint="/store"}

# Request duration p95
histogram_quantile(0.95,
  rate(openmemory_request_duration_seconds_bucket{client="client_123"}[5m])
)
```

### Database Metrics (postgres_exporter)

```promql
# Total memories stored per client
pg_openmemory_stored_memories{client="user_123"}

# Memories per sector per client
pg_openmemory_stored_memories_by_sector{client="user_123", sector="semantic"}
```

## Development Workflow

### Local Testing

```bash
# Start stack
./deploy.sh --target local

# Test OpenMemory API
curl -X POST http://localhost:8080/api/store \
  -H "Authorization: Bearer client_123.secret_key" \
  -H "Content-Type: application/json" \
  -d '{"content": "Test memory", "userId": "client_123"}'

# Check metrics
curl http://localhost:9091/metrics | grep openmemory

# View logs
./deploy.sh --target local --action logs

# Stop
./deploy.sh --target local --action down
```

### Updating Helm Chart

When you modify `docker-compose.yml`:

```bash
# Regenerate k8s manifests
kompose convert -f docker-compose.yml -o charts/cybermem/templates/

# Test locally with k3d
k3d cluster create test
helm install cybermem charts/cybermem -f charts/cybermem/values-local.yaml

# Cleanup
k3d cluster delete test
```

## Production Deployment

### VPS Setup (Hetzner CX22 example)

```bash
# 1. Provision VPS (Terraform - future)
cd terraform
terraform apply

# 2. Install k3s
ssh root@vps.example.com
curl -sfL https://get.k3s.io | sh -

# 3. Get kubeconfig
scp root@vps.example.com:/etc/rancher/k3s/k3s.yaml ~/.kube/config-vps
export KUBECONFIG=~/.kube/config-vps

# 4. Install Helm chart
./deploy.sh --target vps
```

### Secrets Management

⚠️ **IMPORTANT:** Update secrets before production deployment!

```bash
# Edit values-vps.yaml
nano charts/cybermem/values-vps.yaml

# Replace:
# - openmemory.config.apiKey
# - openmemory.config.embeddings.openaiApiKey
# - postgres.config.password
# - grafana.adminPassword

# Or use Sealed Secrets (recommended)
kubectl create secret generic cybermem-secrets \
  --from-literal=om-api-key='...' \
  --from-literal=openai-api-key='...' \
  -n cybermem --dry-run=client -o yaml | \
  kubeseal -o yaml > charts/cybermem/templates/sealed-secret.yaml
```
